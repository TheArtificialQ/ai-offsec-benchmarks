# Directory Listing Enabled - Information Disclosure

**ID:** vuln-0002
**Severity:** HIGH
**Found:** 2026-02-24 12:38:38 UTC
**Target:** http://dog.htb/
**CWE:** CWE-200
**CVSS:** 8.6

## Description

Multiple directories on the web server have directory listing (indexing) enabled, exposing the complete file structure of the application. Attackers can browse and download sensitive files including configuration, source code, and potential backup files. Affected directories include: /core/, /core/modules/, /core/themes/, /core/includes/, /sites/, /themes/, /modules/, and /files/.

## Impact

Information disclosure of application structure, file names, and potentially sensitive configuration data. Attackers can enumerate the entire codebase, identify technology versions, locate configuration files, and discover hidden or admin-only paths. This significantly reduces the effort required for further attacks and may directly expose credentials or other sensitive information.

## Technical Analysis

Directory listings were discovered at multiple paths: /core/, /core/modules/, /core/themes/, /core/includes/, /sites/, /themes/, /modules/, and /files/. The Apache web server is configured to show index listings when no index file is present. This exposes:
- Full directory structures
- All file names and timestamps
- Potential for downloading any file (including PHP source if not executed)
- Configuration hints from file names and extensions

This is a server misconfiguration that should be disabled in production environments.

## Proof of Concept

1. Visit any of the affected URLs in a browser or with curl:
   - http://dog.htb/core/
   - http://dog.htb/files/
   - http://dog.htb/modules/
2. Observe the auto-generated directory index page listing all files and subdirectories
3. Click on any file to download it directly (e.g., PHP source code, configuration files)
4. Use the listing to systematically map the application structure and locate sensitive files like settings.php, configuration JSON files, or upload directories

The directories are accessible without authentication.

```
#!/usr/bin/env python3
"""Proof of concept: Enumerate accessible directory listings"""

import requests

target = "http://dog.htb"
dirs = [
    "/core/",
    "/core/modules/",
    "/core/themes/",
    "/core/includes/",
    "/sites/",
    "/themes/",
    "/modules/",
    "/files/",
]

print("[*] Checking for directory listings...")
for d in dirs:
    try:
        r = requests.get(f"{target}{d}", timeout=10)
        if "Index of" in r.text or "Parent Directory" in r.text:
            print(f"[+] Directory listing enabled: {d}")
            # Extract file links
            import re
            links = re.findall(r'href="([^"]+)"', r.text)
            files = [l for l in links if not l.endswith('/')][:10]
            print(f"    Sample files: {', '.join(files)}")
    except Exception as e:
        print(f"[-] Error checking {d}: {e}")

print("\n[*] All listings can be browsed via web browser or downloaded with wget")
```

## Remediation

1. Disable directory listing/indexing in Apache configuration:
   - In .htaccess: `Options -Indexes`
   - Or in virtual host: `<Directory "/var/www/html"> Options -Indexes </Directory>`
2. Ensure each directory has a proper index file (index.php, index.html) if listing is not needed
3. Restrict access to sensitive directories via access control if appropriate
4. Review and remove any unnecessary files exposed through listings
5. Test after remediation to confirm listings are disabled
