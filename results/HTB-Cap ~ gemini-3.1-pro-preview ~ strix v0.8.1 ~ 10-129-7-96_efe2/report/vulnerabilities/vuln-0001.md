# Insecure Direct Object Reference (IDOR) Leading to Information Disclosure in /data/ Endpoint

**ID:** vuln-0001
**Severity:** HIGH
**Found:** 2026-02-28 14:26:41 UTC
**Target:** http://cap.htb
**Endpoint:** /data/{id}
**Method:** GET
**CWE:** CWE-639
**CVSS:** 7.5

## Description

The application exposes an endpoint at `/data/` which is vulnerable to Insecure Direct Object Reference (IDOR). By manipulating the integer value in the URL path (e.g., from `/data/1` to `/data/0`), an attacker can access packet capture files belonging to other users or system processes. Specifically, accessing `/data/0` allows the download of a PCAP file containing sensitive information.

## Impact

An attacker can access and download sensitive packet capture files. In this specific instance, analyzing the downloaded PCAP file (`/data/0`) revealed plaintext FTP credentials (username: `nathan`, password: `Buck3tH4TF0RM3!`). These credentials were subsequently used to gain SSH access to the system, leading to further compromise.

## Technical Analysis

The vulnerability stems from the lack of proper authorization checks on the `/data/<id>` endpoint. The application uses sequential integers to identify resources (PCAP files) but does not verify if the currently authenticated user (or unauthenticated user, as in this case) has the necessary permissions to access the requested resource. When `/data/0` is requested, the application blindly serves the corresponding data, which happens to be a capture containing sensitive FTP traffic.

## Proof of Concept

1. Navigate to `http://cap.htb/data/0`.
2. Observe the download button and the associated link `/download/0`.
3. Download the PCAP file using a tool like `curl`: `curl -O http://cap.htb/download/0`.
4. Analyze the downloaded PCAP file using `tcpdump` or Wireshark: `tcpdump -r 0 -A | grep -i pass`.
5. Observe the plaintext FTP credentials in the output.

```
import requests

url = "http://cap.htb/download/0"
response = requests.get(url)

if response.status_code == 200:
    with open("0.pcap", "wb") as f:
        f.write(response.content)
    print("Successfully downloaded PCAP file. Please analyze '0.pcap' for credentials.")
else:
    print(f"Failed to download PCAP file. Status code: {response.status_code}")
```

## Remediation

1. Implement strict authorization checks on the `/data/<id>` and `/download/<id>` endpoints.
2. Ensure that users can only access resources they have explicitly been granted permission to view.
3. Consider using unpredictable identifiers (e.g., UUIDs) instead of sequential integers to make enumeration more difficult.
4. Avoid storing or transmitting sensitive credentials in plaintext.
