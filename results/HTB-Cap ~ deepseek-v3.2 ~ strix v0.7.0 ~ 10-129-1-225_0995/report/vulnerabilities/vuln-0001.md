# OpenSSH 8.2p1 Vulnerable to Terrapin Attack (CVE-2023-48795) with Weak Algorithm Configuration

**ID:** vuln-0001
**Severity:** HIGH
**Found:** 2026-02-21 16:45:54 UTC
**Target:** 10.129.1.225:22
**Endpoint:** TCP port 22
**Method:** SSH protocol negotiation
**CVE:** CVE-2023-48795
**CVSS:** 7.3

## Description

The SSH service on port 22 (OpenSSH 8.2p1 Ubuntu 4ubuntu0.2) was found to be vulnerable to the Terrapin attack (CVE-2023-48795) and configured with multiple weak cryptographic algorithms.

The server accepts the vulnerable chacha20-poly1305@openssh.com cipher, which allows an attacker to truncate protected messages in the SSH protocol. Additionally, the server configuration includes several weak and deprecated algorithms that reduce the overall security posture.

The ssh-audit tool confirmed the presence of the vulnerable cipher and identified multiple configuration weaknesses including suspected NSA-backdoored elliptic curves, broken SHA-1 hash algorithms, insufficient connection throttling, and enabled compression.

## Impact

Successful exploitation of the Terrapin attack could allow an attacker to:
- Truncate protected SSH messages, potentially bypassing security controls
- Bypass keystroke timing obfuscation features
- Weaken the integrity and confidentiality of SSH connections
- Combine with other attacks to compromise SSH sessions

The weak algorithm configuration further reduces security by:
- Enabling algorithms with known weaknesses (SHA-1, small key sizes)
- Using cryptographic curves suspected of being backdoored
- Potentially allowing downgrade attacks
- Enabling compression that could lead to CRIME/BREACH style attacks if combined with encryption

## Technical Analysis

The SSH server running OpenSSH 8.2p1 Ubuntu 4ubuntu0.2 was analyzed using ssh-audit version 3.3.0. The analysis revealed:

1. **CVE-2023-48795 (Terrapin Attack) Vulnerability**:
   - Server accepts chacha20-poly1305@openssh.com cipher
   - This cipher is vulnerable to message prefix truncation attacks
   - Attack allows bypassing security features and truncating protected messages

2. **Weak Algorithm Configuration**:
   - ecdh-sha2-nistp256/384/521: Suspected NSA-backdoored elliptic curves
   - ssh-rsa: Uses broken SHA-1 hash algorithm (deprecated in OpenSSH 8.8)
   - ecdsa-sha2-nistp256: Suspected NSA-backdoored curve with weak random number generator
   - hmac-sha1 and hmac-sha1-etm@openssh.com: Broken SHA-1 hash algorithm
   - diffie-hellman-group14-sha256: 2048-bit modulus providing only 112-bit symmetric strength

3. **Configuration Issues**:
   - Insufficient connection throttling (38 connections in 0.874s = 43.5 conns/sec)
   - Vulnerable to DHEat DoS attack (CVE-2002-20001)
   - Compression enabled (zlib@openssh.com) which could enable CRIME/BREACH attacks

4. **Additional Findings**:
   - Server does not use post-quantum key exchange algorithms
   - Warns about "store now, decrypt later" vulnerability to quantum attacks

## Proof of Concept

To verify the vulnerability:

1. Install ssh-audit: `sudo apt-get install ssh-audit`
2. Run audit: `ssh-audit 10.129.1.225`
3. Observe the following in the output:
   - "(enc) chacha20-poly1305@openssh.com -- [warn] vulnerable to the Terrapin attack (CVE-2023-48795)"
   - Multiple [fail] and [warn] ratings for weak algorithms
   - Note about insufficient connection throttling

4. Confirm cipher acceptance:
   ```
   ssh -c chacha20-poly1305@openssh.com -o ConnectTimeout=5 user@10.129.1.225
   ```
   (Server will accept connection negotiation with vulnerable cipher)

The vulnerability is confirmed by the presence of the chacha20-poly1305@openssh.com cipher in the server's accepted algorithms list as reported by ssh-audit.

#!/usr/bin/env python3
"""
SSH Terrapin Attack & Weak Configuration Verification Script
Target: 10.129.1.225:22
"""

import subprocess
import sys

def run_ssh_audit(target):
    """Run ssh-audit and check for Terrapin vulnerability"""
    print(f"Running ssh-audit on {target}...")
    
    try:
        result = subprocess.run(
            ["ssh-audit", target],
            capture_output=True,
            text=True,
            timeout=30
        )
        
        if result.returncode == 0:
            output = result.stdout
            
            # Check for Terrapin vulnerability
            if "vulnerable to the Terrapin attack (CVE-2023-48795)" in output:
                print("✓ VULNERABLE to Terrapin Attack (CVE-2023-48795)")
                
                # Extract vulnerable cipher line
                for line in output.split('\n'):
                    if "chacha20-poly1305@openssh.com" in line and "Terrapin" in line:
                        print(f"  Vulnerable cipher: {line.strip()}")
            else:
                print("✗ Not vulnerable to Terrapin Attack")
            
            # Check for weak algorithms
            weak_algorithms = []
            for line in output.split('\n'):
                if "[fail]" in line or "[warn]" in line:
                    weak_algorithms.append(line.strip())
            
            if weak_algorithms:
                print(f"\nFound {len(weak_algorithms)} weak/deprecated algorithms:")
                for alg in weak_algorithms[:10]:  # Show first 10
                    print(f"  {alg}")
            
            return output
        else:
            print(f"Error running ssh-audit: {result.stderr}")
            return None
            
    except subprocess.TimeoutExpired:
        print("ssh-audit timed out")
        return None
    except FileNotFoundError:
        print("ssh-audit not installed. Install with: sudo apt-get install ssh-audit")
        return None

def test_cipher_acceptance(target, cipher):
    """Test if server accepts specific cipher"""
    print(f"\nTesting if server accepts cipher: {cipher}")
    
    try:
        cmd = [
            "ssh", "-o", "StrictHostKeyChecking=no",
            "-o", "ConnectTimeout=5",
            "-c", cipher,
            f"user@{target}",
            "echo 'test' 2>/dev/null"
        ]
        
        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=10
        )
        
        # If we get "Permission denied", cipher was accepted but auth failed
        # If we get other error, cipher might not be accepted
        if "Permission denied" in result.stderr:
            print(f"  ✓ Server accepts cipher: {cipher}")
            return True
        elif result.returncode == 0:
            print(f"  ✓ SUCCESS - Authenticated with cipher: {cipher}")
            return True
        else:
            print(f"  ✗ Server may not accept cipher: {result.stderr[:200]}")
            return False
            
    except subprocess.TimeoutExpired:
        print(f"  ⏱️  Timeout testing cipher: {cipher}")
        return False
    except Exception as e:
        print(f"  ✗ Error: {str(e)}")
        return False

def main():
    target = "10.129.1.225"
    
    print(f"SSH Security Assessment for {target}:22")
    print("=" * 60)
    
    # Run ssh-audit
    audit_output = run_ssh_audit(target)
    
    if audit_output:
        # Test vulnerable cipher acceptance
        test_cipher_acceptance(target, "chacha20-poly1305@openssh.com")
    
    print("\n" + "=" * 60)
    print("Assessment Complete")
    
    # Summary
    print("\nSUMMARY:")
    print("-" * 40)
    print("Vulnerability: CVE-2023-48795 (Terrapin Attack)")
    print("Status: CONFIRMED via ssh-audit")
    print("Impact: Message truncation, security feature bypass")
    print("Affected Cipher: chacha20-poly1305@openssh.com")
    print("\nRemediation:")
    print("1. Update OpenSSH to version 9.6 or later")
    print("2. Disable chacha20-poly1305@openssh.com cipher")
    print("3. Remove weak algorithms identified by ssh-audit")

if __name__ == "__main__":
    main()

## Remediation

1. **Immediate Actions**:
   - Update OpenSSH to version 9.6 or later which includes protection against Terrapin attacks
   - If updating is not possible, modify sshd_config to disable vulnerable ciphers:
     ```
     Ciphers -chacha20-poly1305@openssh.com
     MACs -*-etm@openssh.com
     ```

2. **Algorithm Hardening**:
   - Remove weak and deprecated algorithms from sshd_config:
     ```
     KexAlgorithms -ecdh-sha2-nistp256,ecdh-sha2-nistp384,ecdh-sha2-nistp521
     HostKeyAlgorithms -ssh-rsa,-ecdsa-sha2-nistp256
     MACs -hmac-sha1,-hmac-sha1-etm@openssh.com
     ```
   - Prefer modern algorithms:
     ```
     KexAlgorithms curve25519-sha256,curve25519-sha256@libssh.org,diffie-hellman-group16-sha512,diffie-hellman-group18-sha512
     HostKeyAlgorithms ssh-ed25519,rsa-sha2-512,rsa-sha2-256
     ```

3. **Connection Throttling**:
   - Implement connection rate limiting in sshd_config:
     ```
     MaxStartups 10:30:60
     PerSourceMaxStartups 2
     ```

4. **Compression**:
   - Disable compression if not needed:
     ```
     Compression no
     ```

5. **Post-Quantum Considerations**:
   - Monitor OpenSSH releases for post-quantum cryptography support
   - Plan migration to post-quantum secure algorithms when available

6. **Verification**:
   - After changes, run ssh-audit to verify vulnerabilities are mitigated
   - Test SSH connectivity from all required client systems
   - Monitor logs for any connection issues
