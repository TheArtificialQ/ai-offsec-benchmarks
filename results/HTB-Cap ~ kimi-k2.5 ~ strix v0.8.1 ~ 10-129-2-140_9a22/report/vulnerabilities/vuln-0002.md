# Unauthenticated Network Information Disclosure via System Command Endpoints

**ID:** vuln-0002
**Severity:** MEDIUM
**Found:** 2026-02-22 15:18:44 UTC
**Target:** 10.129.2.140
**Endpoint:** /ip, /netstat, /capture
**Method:** GET
**CWE:** CWE-200
**CVSS:** 5.3

## Description

The Security Dashboard application at http://10.129.2.140 exposes sensitive system information through multiple endpoints that execute system commands without requiring authentication:

1. **`/ip` endpoint** - Executes `ifconfig` command and displays complete network interface configuration including:
   - IP addresses (10.129.2.140)
   - MAC addresses
   - Network masks
   - Interface statistics

2. **`/netstat` endpoint** - Executes `netstat` command and displays:
   - Active network connections
   - Listening ports
   - Process IDs and program names
   - Connection states

3. **`/capture` endpoint** - Executes `tcpdump` to capture network traffic

All endpoints are accessible without authentication, providing valuable reconnaissance information to potential attackers.

## Impact

Information disclosure enables attackers to:
- Map the internal network topology
- Identify running services and their ports
- Gather IP addressing schemes for lateral movement planning
- Identify active connections to other systems
- Discover potential pivot points for further attacks

This information significantly aids in planning and executing follow-on attacks against the target system and potentially the broader network.

## Technical Analysis

The vulnerability exists because the application:

1. **Lacks Authentication**: The `/ip`, `/netstat`, and `/capture` endpoints do not implement any authentication or authorization checks.

2. **Executes System Commands**: These endpoints directly execute system commands (`ifconfig`, `netstat`, `tcpdump`) and return raw output to the user.

3. **Exposes Sensitive Output**: The command output includes:
   - Interface names and hardware addresses (from ifconfig)
   - Connection tables showing remote hosts and services (from netstat)
   - Network traffic captures (from tcpdump)

The exposed data reveals:
- System IP: 10.129.2.140
- Netmask: 255.255.0.0
- Gateway connections
- Active SSH connections
- DNS queries (127.0.0.53:53)
- All listening services

## Proof of Concept

1. Access the `/ip` endpoint without authentication:
   ```
   curl http://10.129.2.140/ip
   ```
   Observe full `ifconfig` output including IP addresses, MAC addresses, and interface statistics.

2. Access the `/netstat` endpoint without authentication:
   ```
   curl http://10.129.2.140/netstat
   ```
   Observe active connections, listening ports, and process information.

3. Access the `/capture` endpoint to trigger traffic capture:
   ```
   curl http://10.129.2.140/capture
   ```
   The endpoint initiates a 5-second tcpdump capture.

```
#!/usr/bin/env python3
"""
Network Information Disclosure PoC
Demonstrates unauthenticated access to system information
"""

import requests
import re

TARGET = "10.129.2.140"
BASE_URL = f"http://{TARGET}"

def test_info_disclosure():
    """Test for information disclosure vulnerabilities"""
    print(f"[*] Testing target: {TARGET}")
    
    # Test /ip endpoint
    print("\n[+] Testing /ip endpoint...")
    resp = requests.get(f"{BASE_URL}/ip", timeout=10)
    if resp.status_code == 200:
        print("[!] VULNERABLE: /ip accessible without authentication")
        # Extract IP address
        ip_match = re.search(r'inet (\d+\.\d+\.\d+\.\d+)', resp.text)
        if ip_match:
            print(f"    Found IP: {ip_match.group(1)}")
    
    # Test /netstat endpoint
    print("\n[+] Testing /netstat endpoint...")
    resp = requests.get(f"{BASE_URL}/netstat", timeout=10)
    if resp.status_code == 200:
        print("[!] VULNERABLE: /netstat accessible without authentication")
        # Extract listening ports
        ports = re.findall(r':(\d+)\s+.*LISTEN', resp.text)
        if ports:
            print(f"    Found listening ports: {', '.join(set(ports))}")
    
    # Test /capture endpoint
    print("\n[+] Testing /capture endpoint...")
    resp = requests.get(f"{BASE_URL}/capture", timeout=10)
    if resp.status_code == 200:
        print("[!] VULNERABLE: /capture accessible without authentication")
        print("    Can trigger network traffic capture")
    
    print("\n[*] Assessment: MEDIUM risk information disclosure")
    print("    Recommend: Implement authentication for system endpoints")

if __name__ == "__main__":
    test_info_disclosure()
```

## Remediation

1. **Implement Authentication**: Require valid user authentication before allowing access to `/ip`, `/netstat`, and `/capture` endpoints.

2. **Implement Authorization**: Ensure only authorized users (e.g., administrators) can access system information endpoints.

3. **Sanitize Output**: Filter or redact sensitive information from command output before displaying to users.

4. **Alternative Architecture**: Instead of executing system commands directly, consider:
   - Using a privileged service to gather sanitized statistics
   - Implementing rate limiting to prevent abuse
   - Logging all access for security monitoring

5. **Network Segmentation**: If these endpoints must remain unauthenticated, consider restricting access to trusted networks only.
