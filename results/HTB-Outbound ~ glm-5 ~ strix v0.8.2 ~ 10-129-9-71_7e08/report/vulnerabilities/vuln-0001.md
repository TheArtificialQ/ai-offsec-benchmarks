# Roundcube Post-Auth RCE via PHP Object Deserialization (CVE-2025-49113)

**ID:** vuln-0001
**Severity:** HIGH
**Found:** 2026-03-03 20:08:33 UTC
**Target:** http://mail.outbound.htb
**Endpoint:** /?_task=settings&_action=upload</endpoint>
<parameter=method>GET
**CVE:** CVE-2025-49113
**CWE:** CWE-502
**CVSS:** 8.8

## Description

A critical remote code execution vulnerability was identified in Roundcube Webmail version 1.6.10. The vulnerability exists in the handling of the `_from` parameter during settings upload functionality, allowing authenticated attackers to inject serialized PHP objects that lead to arbitrary command execution.

The vulnerability was successfully exploited to gain remote code execution on the target server as the www-data user. Through this access, sensitive credentials were extracted from Roundcube's database sessions table, including encrypted passwords that were subsequently decrypted using Roundcube's des_key configuration value.

## Impact

This vulnerability allows any authenticated user of Roundcube Webmail to execute arbitrary commands on the underlying server. This leads to:

1. Complete Server Compromise - Attackers can execute any command as the web server user
2. Credential Theft - Access to Roundcube's session database reveals encrypted IMAP passwords that can be decrypted
3. Lateral Movement - Stolen credentials enable pivoting to other user accounts on the system
4. Data Breach - Access to all emails, contacts, and sensitive data stored in Roundcube

## Technical Analysis

The vulnerability exploits improper input validation in the `_from` parameter handling. The attack chain:

1. Authentication: Login to Roundcube with valid credentials to obtain session
2. Payload Construction: Create a serialized `Crypt_GPG_Engine` PHP object with malicious command in the `_gpgconf` property
3. Injection: Send crafted GET request to settings upload endpoint with serialized object in `_from` parameter
4. Deserialization: Roundcube deserializes the object without proper validation
5. Execution: The `Crypt_GPG_Engine` destructor triggers command execution via the injected `_gpgconf` value

The exploit uses PHP object gadget chain where the serialized object contains:
- Class: `Crypt_GPG_Engine`
- Property `_gpgconf`: Contains shell command to execute
- When deserialized and destroyed, executes the command

## Proof of Concept

1. Authenticate to Roundcube with valid credentials
2. Create serialized PHP payload using Crypt_GPG_Engine class
3. URL-encode the serialized payload
4. Send GET request to: /?_from=edit-{serialized_payload}&_task=settings&_framed=1&_remote=1&_action=upload
5. The command executes server-side as www-data user

## Remediation

1. Upgrade Roundcube to version 1.6.11 or later immediately
2. Implement input validation on the _from parameter
3. Disable object deserialization for user-controlled input
4. Rotate all credentials and regenerate des_key
5. Deploy WAF rules to detect serialization payloads
